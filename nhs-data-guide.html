NHS Air Quality Data Integration Guide

How to populate the three-panel air quality widget (Long-term · Recent · Forecast) from the exposure service and London Air forecast. Written for NHS implementers integrating with Cerner or similar EHR systems.

Exposure service API

The exposure service returns hourly pollutant concentrations for a geocoded home location. A single range request serves both the Long-term and Recent panels — each hourly record includes two concentrations per species. The API does not return a pre-shaped widget payload; your integration layer maps the fields below into the widget shape.

Patient location

Geocode the patient’s home address to latitude and longitude. The API takes lat and lng — not a postcode. Postcode is for display in the widget header only.

Example: Eleanor Marsh, 14 Walworth Road, Southwark, London SE1 6EE → lat=51.4933, lng=-0.0995.

Single hour (legacy) https://swift-exposure.nw.r.appspot.com/exposure/london/coord key, lat, lng, species, timestamp, weighted=true

Use the range endpoint (/coords) for NHS integration. Dates must be ISO 8601 UTC strings (e.g. 2026-07-03T00:00:00Z) — the API does not accept -3d shorthand. Pass the API key as a key query parameter.

Fields on each hourly record

Each hourly object includes:

  • timestamp — start of the hour in GMT/UTC
  • data_map_start — base year of the annual-mean map used for value, as an ISO datetime string, e.g. 2022-01-01T00:00:00.000+0000. Constant for a given response; use its calendar year for the Long-term panel label (annualYear)
  • results[] — per-species concentrations (see below)

Two fields per species

Each entry in results[] includes both:

  • value — annual mean at the patient’s lat/lng from the map identified by data_map_start (µg/m³). Constant for a given location regardless of hour; use for the Long-term panel
  • nowcast_value — real-time nowcast concentration for that hour (µg/m³). Varies hour by hour; use for the Recent panel

What to fetch

One range call can cover both panels, but it must start at least seven hours before UK-local 00:00 on day −3. Those preceding hours are needed to calculate the rolling 8-hour O₃ result at 00:00. Convert that local boundary with the IANA timezone Europe/London, then send the resulting instant to the API as UTC. UK midnight is 00:00Z during GMT and 23:00Z on the preceding UTC date during BST. Fetch through the latest complete hour (or through the end of today if the service supplies it). For very long ranges, fetch in weekly chunks to avoid timeouts.

Each timestamp marks the start of the hour in GMT/UTC. Map species to pm25, pm10, no2, o3. Use nowcast_value for Recent hourly series; use value for Long-term annual concentrations (same at every hour for a given location). Derive annualYear from data_map_start — for example 2022-01-01T00:00:00.000+0000 → 2022 — rather than hard-coding the year.

Mapping API response → widget data

The prototype’s mockPatientExposure() in js/air-quality.js shows the widget’s expected shape, not the raw API. Your integration layer must transform the hourly array into:

  • annual + annualYear — read value per species from any hour in the response (constant for a given lat/lng). Set annualYear from data_map_start (calendar year of that datetime — currently 2022 when the field is 2022-01-01T00:00:00.000+0000)
  • recentDays[] — group hours by UK-local calendar day using nowcast_value, retaining each timestamp rather than assuming a fixed 24-element array (a daylight-saving transition day has 23 or 25 hours). Then derive pollutant-specific DAQI inputs: PM₂.₅/PM₁₀ daily means; NO₂ and O₃ keep timestamped hourly series for max-of-hourly / rolling-8h calculations (see DAQI section)
Raw vs widget shape:
  • API returns: flat array of { timestamp, data_map_start, results[{ species, value, nowcast_value }] } in GMT
  • Widget expects: { annual, annualYear, recentDays[{ offset, date, hourly, daily }] } grouped by UK calendar day

Calculating DAQI banding (Recent panel)

Normative official references: Current GOV.UK DAQI pollutant concentration table for concentration ranges and averaging periods, and DEFRA’s Update on Implementation of the DAQI (April 2013) (PDF) for implementation detail on data capture, final rounding, retrospective calculations and triggers. Each pollutant uses a different averaging period; a historical daily summary is not a 24-hour mean for every pollutant.

The concentration thresholds below come directly from the current gov.uk table. Do not use the prototype or a third-party implementation as the normative source.

Step 1 — Pollutant-specific concentrations for DAQI

DAQI is calculated per pollutant using the averaging window defined for that pollutant on gov.uk. Group hourly API values into UK-local calendar days first (see timestamps section below), then apply the rules below. For completed past days (−3d, −2d, −1d), these rules apply directly. For today (the current UK calendar day), see the separate subsection below — PM₂.₅, PM₁₀, and O₃ require additional logic beyond these standard rules.

PM₂.₅ and PM₁₀ — 24-hour daily mean

For each completed UK calendar day, average the unrounded hourly means from UK-local midnight to midnight (time beginning). A normal day has 24 hours; the spring DST transition day has 23 and the autumn transition day has 25. Select hours by timestamp and local-day boundary, so the skipped or repeated clock hour is handled correctly.

pmDailyMean = sum(valid unrounded hourly means in local day) / count(valid hourly means)

DEFRA requires at least 75% data capture for a daily mean (the 2013 guidance gives 18 hourly means for a normal 24-hour day). Applied to the actual DST-day duration, require at least 18 of 23 hours in spring and 19 of 25 hours in autumn. Do not silently calculate an official DAQI value below the required capture; mark it unavailable. The official document does not spell out the 23/25-hour arithmetic, so this is the direct application of its 75% rule to the UK-local interval.

The daily PM index is derived from this mean in Step 2 (round once, then perform the threshold lookup).

NO₂ — maximum hourly index (not a daily mean)

DAQI for NO₂ is based on hourly mean concentrations — do not compute a 24-hour mean and do not reduce the calculation to a single 200 µg/m³ test. For a completed-day summary, keep each valid hourly mean unrounded until its final comparison, round it once (Step 2), map that rounded concentration to the full index 1–10 table, then take the maximum of those hourly indices across the UK-local day. DEFRA’s 75% capture rule requires an hourly mean derived from sub-hourly measurements to contain at least three valid 15-minute means; the exposure API already supplies hourly values, so their upstream completeness should be confirmed with the service owner.

no2DayIndex = max( daqiIndex(roundOnce(hourlyNO2[h])) for valid hours in the local day )

Boundary condition: after final integer rounding, 200 µg/m³ is still Low/index 3. A rounded concentration of at least 201 µg/m³ (≥201) is Moderate or above, and 201 µg/m³ is the first Moderate/index 4 value.

O₃ — rolling 8-hour mean

DAQI for O₃ uses a rolling 8-hour mean at each hour: the mean of the current hour and the previous seven elapsed hourly slots (eight hours total). Compute this from 00:00 on the −3 day. For each completed UK calendar day, take the maximum valid rolling-8h index among windows ending within that local midnight-to-midnight interval. Current-day presentation is addressed separately below.

rolling8h[h] = mean( unrounded hourlyO3[h−7] … hourlyO3[h] ) o3DayIndex = max( daqiIndex(roundOnce(rolling8h[h])) for valid windows ending in that local day )

A rolling window is valid with at least six of its eight hourly means (75% capture), according to DEFRA’s 2013 guidance. At UK-local 00:00, the window includes the prior seven elapsed hours, which may fall on the previous UK calendar day. Request at least seven hours before UK-local midnight on −3d; no additional “BST margin” is needed if the boundary is converted correctly with Europe/London. Assign each rolling result to the local date of the hour at which its window ends.

Today (current day)

The −3d, −2d, and −1d columns are completed UK calendar days only. Use the Step 1 rules above (full PM midnight-to-midnight means; NO₂ max hourly index; O₃ max rolling-8h index). Do not apply triggers retrospectively to those days.

The Today column shows the current situation for each pollutant, then takes the maximum across pollutants (Step 3). “Current” means different things per pollutant: NO₂ follows the latest hour only; PM₂.₅, PM₁₀ and O₃ use predictive triggers that can remain in force after they fire, optionally superseded by a higher measurement-based index.

What “current” means per pollutant

  • NO₂ — index of the latest complete UK-local hour only. A high hour earlier today (e.g. ≥201 µg/m³ three hours ago) does not keep Today elevated. No separate trigger table.
  • PM₂.₅ / PM₁₀ — from UK-local midnight, run the trigger system on consecutive hourly means. An activated trigger index stays in force until it is superseded by a higher partial-day mean index (from 19:00 onward — see below) or until UK-local midnight resets the day. A later fall in hourly concentrations does not by itself clear the trigger.
  • O₃ — same trigger persistence idea, but the measurement that can supersede is the current rolling 8-hour mean index (only if higher than the active trigger). Clear the O₃ trigger after about four hours without a continuing trigger condition, or at UK-local midnight. If the rolling 8-hour index is higher than the trigger, use the rolling index.

How a trigger activates

A trigger uses two consecutive hourly means (time beginning), biased to rising concentrations: the first hour is greater than or equal to the trigger threshold for an index level, and the second hour is greater than or equal to the first. When that happens, that index level becomes the active trigger for the pollutant (and may step up if a higher index trigger fires later). Triggers are for real-time Today information only — never for −3d/−2d/−1d.

Sources for trigger thresholds

DEFRA’s Update on Implementation of the DAQI (April 2013) publishes official band triggers only (Moderate or above, High or above, Very High or above). Those band anchors are: PM₁₀ = 68, 107, 177 µg/m³; PM₂.₅ = 50, 74, 101 µg/m³; O₃ = 105, 170 µg/m³, and Very High for O₃ not determined.

DEFRA commissioned Environmental Research Group (ERG), King’s College London, to develop those band triggers. ERG then extended the same method to produce per–index-point (1–10) triggers so a Today ladder can resolve individual DAQI levels, not only broad bands. The index-point table below is that ERG extension. Treat it as proprietary implementation data for this NHS integration — do not republish it as a public DEFRA specification, and do not invent or interpolate alternative index-point triggers.

PM₂.₅ / PM₁₀ — superseding the trigger with a partial-day mean

From UK-local midnight (use Europe/London — during BST, UK midnight is 23:00 UTC on the previous calendar date, not 00:00 GMT), operate on triggers as above.

From UK-local 19:00, enough complete hours exist for a 75% day mean: the mean of hours 00:00 through 18:00 inclusive (19 hourly slots on a normal day). Round once, map to a DAQI index. If that index is strictly greater than the active trigger index, it replaces the trigger as the pollutant’s Today value. If it is lower or equal, keep the trigger. As later hours complete (19:00, 20:00, …), extend the mean from UK midnight through the latest complete hour and apply the same replace-only-if-higher rule. At UK-local midnight, clear triggers and start again.

pmTodayIndex = max( activeTriggerIndex, daySoFarIndex ) // daySoFarIndex only from 19:00; only replaces when higher // activeTriggerIndex persists until replaced or midnight

O₃ — superseding with the rolling 8-hour mean

Compute the current valid rolling 8-hour mean, round once, and map to an index. Today’s O₃ value is the maximum of the active trigger index (if any) and that rolling index. The rolling index replaces the trigger only when it is higher; otherwise the trigger remains until the ~four-hour clear or midnight.

o3TodayIndex = max( activeTriggerIndex, rolling8hIndex )

Combining pollutants for Today

todayLevel = max( pm25TodayIndex, pm10TodayIndex, no2LatestHourIndex, o3TodayIndex )

Timestamps and UK calendar days

The exposure service returns hourly records where each timestamp is the start of that hour, always in GMT (UTC). The widget, however, labels days as UK calendar dates (−3d, −2d, −1d, Today) — midnight to midnight in UK local time.

During BST (British Summer Time, roughly late March to late October), UK local time is UTC+1. A single UK calendar day therefore spans two GMT dates. You must not group records by GMT date alone.

BST example — UK 6 July 2026 (a summer day):

UK-local 6 Jul 00:00–23:00 corresponds to these GMT timestamps (time beginning):

  • UK 00:00 → 2026-07-05T23:00:00Z
  • UK 01:00 → 2026-07-06T00:00:00Z
  • …
  • UK 23:00 → 2026-07-06T22:00:00Z

To compute the PM daily mean for “6 Jul”, select exactly these 24 hourly records — not all records where the UTC date is 6 Jul. The same local-day selection applies when taking NO₂ hourly maxima or O₃ rolling-8h maxima for that day. For O₃ at UK 00:00 on 6 Jul, the rolling window also needs UK-local hours from 17:00–23:00 on 5 Jul — i.e. 2026-07-05T16:00:00Z through 2026-07-05T22:00:00Z during BST.

Implementation checklist:

  1. Parse each API timestamp as a UTC instant and convert it to UK local time using Europe/London, which handles GMT, BST and transition days automatically.
  2. Assign each hour to a UK calendar date based on its local time.
  3. For each completed UK date, collect all timestamps in its local midnight-to-midnight interval (23, 24 or 25 hours). PM₂.₅/PM₁₀: daily mean with at least 75% capture. NO₂: max hourly index for the historical-day summary. O₃: max valid rolling-8h index, using prior-day hours where needed and at least six values per window.
  4. Set from to the UTC instant exactly seven hours before UK-local 00:00 on −3d. Set to to the latest complete hour needed. Do not append a guessed one-day BST buffer.
  5. Handle Today separately using the trigger and supersede rules in the Today section above (NO₂ = latest hour only; PM/O₃ triggers persist until superseded or cleared).

Outside BST (GMT = UK local), UK midnight aligns with GMT midnight and grouping is straightforward — but always use UK-local assignment so the logic works year-round.

Step 2 — Round, then map to DAQI index

Before comparing a concentration to DAQI levels, round it once to an integer. DEFRA’s 2013 guidance is explicit that rounding is the last step: calculate hourly, rolling 8-hour and daily means from unrounded inputs, then round once immediately before the index comparison. Do not pre-round hourly inputs used in a longer mean and do not round via an intermediate decimal place.

Prescribed rounding evidence and implementation note:
  • The official example says 50.486 → 50 directly; it warns that first producing 50.5 and then rounding again would incorrectly produce 51.
  • DEFRA’s document requires one final integer rounding step but does not specify how exact x.5 ties are handled. The NHS integration project’s explicit implementation convention is that exact ties round upward: 50.5 → 51. This is a project decision, not a rule attributed to the DEFRA document.
  • DAQI concentrations are nonnegative, so a safe cross-language expression is floor(value + 0.5) at the final comparison only. In JavaScript, use Math.floor(value + 0.5); do not generalise JavaScript’s Math.round() behaviour to negative numbers.

Compare each rounded value against the lower-bound thresholds for that pollutant. The index is the highest band whose threshold the rounded value meets or exceeds (levels 1–10). Gov.uk publishes these as integer ranges (e.g. O₃ level 3 = 67–100, level 4 = 101–120); the lower-bound table below is equivalent.

Step 3 — Take the highest index across pollutants

For each completed day, the displayed DAQI level is the maximum index across PM₂.₅, PM₁₀, NO₂, and O₃ — using each pollutant’s Step 1 result (PM daily mean; NO₂ max hourly; O₃ max rolling-8h). For Today, derive each pollutant’s current index as in the Today section (NO₂ latest hour; PM/O₃ active trigger with optional higher measurement supersede), then take the maximum across available pollutants.

dayLevel = max(pmDayIndex, pm10DayIndex, no2DayIndex, o3DayIndex) // completed days todayLevel = max(pm25TodayIndex, pm10TodayIndex, no2LatestHourIndex, o3TodayIndex)

Long-term bar mathematics (Designs 3.1 / 3.2)

The long-term panel compares annual mean concentrations to WHO air quality guidelines. It uses a separate scale from DAQI — do not mix annual WHO comparison with daily index levels.

WHO guidelines (µg/m³)

PollutantWHO annual guideline
PM₂.₅5
PM₁₀15
NO₂10
O₃60

Bar height

The WHO dotted line is fixed at 52% of the bar track height.

Below the line (concentration up to the WHO guideline):

if annual ≥ WHO: belowHeight = 52% of track else: belowHeight = (annual / WHO) × 52%

Above the line (excess over WHO):

excessRatio = (annual − WHO) / WHO aboveHeight = (excessRatio / maxExcessAcrossPollutants) × 48% of track

The pollutant with the largest excess ratio fills the zone above the line; others scale proportionally.

Display conventions

  • Show annual concentrations as integers (µg/m³)
  • Show ratio as 3.6× — no “WHO” suffix on the multiplier
  • Label the year from annualYear, taken from data_map_start (e.g. 2022)
  • Use pollutant bar colours (solid above WHO, light fill below); names in the key strip below the panel

Panel-by-panel summary

Long-term

  • Data: value per species from the same range call (annual mean at lat/lng from the map identified by data_map_start); set annualYear from that field’s calendar year
  • Render: four vertical bars (PM₂.₅, PM₁₀, NO₂, O₃), shared WHO dotted line at 52%, ratio above bar, concentration below
  • Key strip: pollutant colour swatches + names below the panel

Recent

  • Data: nowcast_value from the same range call; group into UK-local calendar days (−3d through today), adjusting for BST
  • Process: group timestamped data into UK-local days (DST-aware) → for −3d/−2d/−1d use PM daily means, NO₂ max hourly index and O₃ max rolling-8h index, enforcing 75% data capture → round once and compare with the official thresholds → max across pollutants. For Today, use the current-situation rules and ERG index-point trigger table in the Today section (no retrospective triggers on past days).
  • Render: four day columns (−3d, −2d, −1d, Today) with 10-segment DAQI ladders
  • Key strip: DAQI band legend below the panel

Forecast

  • Data: text band from London Air forecast
  • Process: map band to display index (Low→3, Moderate→6, High→9, V.High→10)
  • Render: single DAQI ladder for tomorrow; band name and date below
