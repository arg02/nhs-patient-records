NHS Air Quality Data Integration Guide

How to populate each part of the three-panel widget (Long-term · Recent · Forecast) from the exposure service and London Air forecast. Written for NHS implementers integrating with Cerner or similar EHR systems.

Jump to a panel
  1. 1. Annual average Long-term panel — WHO comparison bars from annual means
  2. 2. Previous days Recent panel −3d / −2d / −1d — completed-day DAQI ladders
  3. 3. Today Recent panel “Today” — triggers + current measurements
  4. 4. Forecast Forecast panel — London Air band for tomorrow

Getting the data (shared)

One exposure-service range request supplies both the Long-term and Recent panels. Forecast comes from a separate London Air page. Geocode the patient’s home to lat / lng (not postcode — postcode is display-only in the header).

Example patient: Eleanor Marsh, SE1 → lat=51.4933, lng=-0.0995.

UseEndpoint
Hourly range (use this) https://swift-exposure-357937791793.europe-west2.run.app/exposure/london/coords
Params: key, lat, lng, species=no2,pm10,pm25,o3, from, to (ISO UTC)

Each hourly record includes:

  • timestamp — start of the hour in GMT/UTC
  • data_map_start — annual-map base date (e.g. 2022-01-01T00:00:00.000+0000) → calendar year for the Long-term label
  • results[] with per species:
    • value — annual mean at lat/lng (constant) → Annual average
    • nowcast_value — hourly nowcast → Previous days and Today
Fetch window for Recent: Start at least seven hours before UK-local midnight on day −3 (needed for the first O₃ rolling 8-hour window). Use IANA timezone Europe/London. During BST, UK midnight is 23:00Z on the previous UTC date. Example for UK days 3–6 Jul 2026: from=2026-07-02T16:00:00Z → to=2026-07-06T22:59:59Z.

Official DAQI references: GOV.UK concentration table · DEFRA April 2013 implementation (PDF)

Bar height maths

The WHO dotted line is fixed at 52% of the bar track.

Below the line (up to the guideline):

if annual ≥ WHO: belowHeight = 52% of track else: belowHeight = (annual / WHO) × 52%

Above the line (excess over WHO):

excessRatio = (annual − WHO) / WHO aboveHeight = (excessRatio / maxExcessAcrossPollutants) × 48% of track

The pollutant with the largest excess fills the zone above the line; others scale proportionally.

Display

  • Concentrations as integers (µg/m³)
  • Ratio as 3.6× (no “WHO” suffix)
  • Solid colour above WHO, light fill below; pollutant names in the key strip

Pipeline

  1. Group nowcast_value hours into UK-local calendar days with Europe/London (handles GMT/BST; a transition day has 23 or 25 hours).
  2. For each completed day, compute a pollutant-specific input (below).
  3. Round once to an integer, then map to DAQI 1–10 (see shared DAQI tables).
  4. dayLevel = max across the four pollutant indices. Render the ladder.

Per-pollutant rules (completed days only)

PM₂.₅ and PM₁₀ — 24-hour daily mean

Average unrounded hourly means from UK-local midnight to midnight (time beginning). Require ≥75% capture (18 of 24 hours on a normal day; 18/23 spring DST; 19/25 autumn DST).

pmDailyMean = sum(valid hours in local day) / count(valid hours)

NO₂ — maximum hourly index

Map each hour to an index, then take the day’s maximum (rather than a 24-hour mean). After rounding, 200 → still Low (3); ≥201 → Moderate (4)+.

no2DayIndex = max( daqiIndex(roundOnce(hourlyNO2[h])) for hours in local day )

O₃ — max of rolling 8-hour means

At each hour: mean of that hour plus the previous seven. Day index = maximum valid window ending in that local day. Window needs ≥6 of 8 hours. Windows at 00:00 need prior-day hours — hence the seven-hour fetch buffer before −3d.

rolling8h[h] = mean(hourlyO3[h−7] … hourlyO3[h]) o3DayIndex = max( daqiIndex(roundOnce(rolling8h[h])) for valid windows in local day )
BST reminder: During summer, UK 00:00 is the previous day’s 23:00Z. Group hours by UK local date rather than by GMT date alone. Example — UK 6 Jul 2026: local 00:00 → 2026-07-05T23:00:00Z … local 23:00 → 2026-07-06T22:00:00Z.

What “current” means

  • NO₂ — index of the latest complete UK-local hour only. A high reading three hours ago leaves the current hour unaffected. NO₂ has no separate trigger table.
  • PM₂.₅ / PM₁₀ — from UK midnight, run triggers on consecutive hours. An activated trigger stays until a higher partial-day mean replaces it (from 19:00) or midnight clears it. Falling hours alone leave the trigger in place.
  • O₃ — same persistence; measurement that can supersede is the current rolling 8-hour index (only if higher). Clear after ~four hours without a continuing trigger, or at midnight.

How a trigger activates

Two consecutive hourly means, rising: first hour ≥ trigger threshold for an index level, second hour ≥ first. Triggers are for Today only; past days (−3d/−2d/−1d) use completed-day statistics instead.

Trigger threshold sources

DEFRA’s April 2013 update publishes official band triggers only (Moderate / High / Very High): PM₁₀ = 68, 107, 177; PM₂.₅ = 50, 74, 101; O₃ = 105, 170 (Very High for O₃ not determined).

DEFRA commissioned Environmental Research Group (ERG), King’s College London, for those band triggers. ERG then extended the method to per–index-point (1–10) triggers so the Today ladder can resolve individual levels. Treat the table below as proprietary for this NHS integration — keep it for implementers rather than republishing it as a public DEFRA specification, and prefer these values over deriving alternatives.

PM — superseding with a partial-day mean

From UK midnight, use triggers. From UK 19:00, mean hours 00:00–18:00 (75% capture), round once, map to an index. Replace the trigger only if that index is higher. Extend the mean as later hours complete. Clear at UK midnight.

pmTodayIndex = max(activeTriggerIndex, daySoFarIndex) // daySoFar from 19:00; replace only if higher

O₃ — superseding with rolling 8-hour

o3TodayIndex = max(activeTriggerIndex, rolling8hIndex)

Combine pollutants

todayLevel = max(pm25TodayIndex, pm10TodayIndex, no2LatestHourIndex, o3TodayIndex)

Rounding

Round once to an integer as the last step before comparing to thresholds. Keep hourly inputs unrounded until that final comparison (avoid rounding intermediate means). Project convention for exact .5 ties: round upward (50.5 → 51). Use Math.floor(value + 0.5) for nonnegative concentrations.
