NHS Air Quality Data Integration Guide

How to populate the three-panel air quality widget (Long-term · Recent · Forecast) from the exposure service and London Air forecast. Written for NHS implementers integrating with Cerner or similar EHR systems.

Exposure service API

The exposure service returns hourly pollutant concentrations for a geocoded home location. A single range request serves both the Long-term and Recent panels — each hourly record includes two concentrations per species. The API does not return a pre-shaped widget payload; your integration layer maps the fields below into the widget shape.

Patient location

Geocode the patient’s home address to latitude and longitude. The API takes lat and lng — not a postcode. Postcode is for display in the widget header only.

Example: Eleanor Marsh, 14 Walworth Road, Southwark, London SE1 6EE → lat=51.4933, lng=-0.0995.

Endpoints

UseBase URLKey parameters
Multi-day / hourly range https://swift-exposure-357937791793.europe-west2.run.app/exposure/london/coords key, lat, lng, species, from, to
Single hour (legacy) https://swift-exposure.nw.r.appspot.com/exposure/london/coord key, lat, lng, species, timestamp, weighted=true

Use the range endpoint (/coords) for NHS integration. Dates must be ISO 8601 UTC strings (e.g. 2026-07-03T00:00:00Z) — the API does not accept -3d shorthand. Pass the API key as a key query parameter.

Two fields per species

Each entry in results[] includes both:

  • value — LAEI 2022 annual mean at the patient’s lat/lng (µg/m³). Constant for a given location regardless of hour; use for the Long-term panel (annual + annualYear: 2022)
  • nowcast_value — real-time nowcast concentration for that hour (µg/m³). Varies hour by hour; use for the Recent panel

What to fetch

One range call from the start of UK-local day −3 through the latest hour covers both panels. Request a wide enough GMT window to cover BST boundaries and at least seven hours before UK 00:00 on −3d for the first O₃ rolling-8h window (see timezone and DAQI sections below). For very long date ranges (e.g. bulk validation), fetch in weekly chunks to avoid timeouts.

Example URL — last 3 days + today

Eleanor Marsh (lat=51.4933, lng=-0.0995), UK calendar days 3–6 July 2026 during BST. The GMT window starts at UK 00:00 on 3 Jul (= 2026-07-02T23:00:00Z):

Each timestamp marks the start of the hour in GMT/UTC. Map species to pm25, pm10, no2, o3. Use nowcast_value for Recent hourly series; use value for Long-term annual concentrations (same at every hour for a given location).

Mapping API response → widget data

The prototype’s mockPatientExposure() in js/air-quality.js shows the widget’s expected shape, not the raw API. Your integration layer must transform the hourly array into:

  • annual + annualYear — read value per species from any hour in the response (constant for a given lat/lng; LAEI 2022 annual mean). Set annualYear to 2022
  • recentDays[] — group hours by UK-local calendar day using nowcast_value; build hourly arrays (indices 0–23 = UK-local 00:00–23:00), then derive pollutant-specific DAQI inputs: PM₂.₅/PM₁₀ daily means; NO₂ and O₃ keep hourly series for max-of-hourly / rolling-8h calculations (see DAQI section)
Raw vs widget shape:
  • API returns: flat array of { timestamp, results[{ species, value, nowcast_value }] } in GMT
  • Widget expects: { annual, annualYear, recentDays[{ offset, date, hourly, daily }] } grouped by UK calendar day

Calculating DAQI banding (Recent panel)

Official reference — DEFRA DAQI thresholds and averaging rules: Pollutant concentrations for the Daily Air Quality Index (DAQI) (gov.uk). Each pollutant uses a different averaging period; the daily index for a pollutant is not always a 24-hour mean.

Thresholds and colours in this prototype match the daqi-vs-caqi reference implementation (colorScales.ts).

Step 1 — Pollutant-specific concentrations for DAQI

DAQI is calculated per pollutant using the averaging window defined for that pollutant on gov.uk. Group hourly API values into UK-local calendar days first (see timestamps section below), then apply the rules below. For completed past days (−3d, −2d, −1d), these rules apply directly. For today (the current UK calendar day), see the separate subsection below — PM₂.₅, PM₁₀, and O₃ require additional logic beyond these standard rules.

PM₂.₅ and PM₁₀ — 24-hour daily mean

For each UK calendar day, average the 24 hourly values for UK-local hours 00:00 through 23:00 (time beginning). Ignore null or missing hours.

pmDailyMean = sum(hourlyValues[00:00..23:00]) / count(validHourlyValues)

The daily PM index is derived from this mean in Step 2 (round, then band lookup).

NO₂ — maximum hourly index (not a daily mean)

DAQI for NO₂ is based on hourly concentrations only — do not compute a 24-hour mean. For each UK-local hour in the day, round the hourly value (Step 2) and map it to a DAQI index. The daily NO₂ index is the maximum hourly index across UK-local 00:00–23:00.

no2DayIndex = max( daqiIndex(round(hourlyNO2[h])) for h in 00:00..23:00 )

O₃ — rolling 8-hour mean

DAQI for O₃ uses a rolling 8-hour mean at each hour: the mean of the current hour and the previous seven consecutive hours (eight hours total). Compute this rolling mean for every UK-local hour from 00:00 on the −3 day through the latest hour on “today”, then for each UK calendar day take the maximum rolling-8h index across that day’s local hours 00:00–23:00.

rolling8h[h] = mean( hourlyO3[h−7] … hourlyO3[h] ) o3DayIndex = max( daqiIndex(round(rolling8h[h])) for h in 00:00..23:00 on that day )

At UK-local 00:00, the rolling window includes the prior seven hours — which may fall on the previous UK calendar day. During BST, UK midnight on the −3 day can require GMT hours from the prior evening (see BST example below). Request enough pre-window hourly data (at least seven hours before UK 00:00 on −3d, plus a BST margin) so the first rolling mean is valid.

Today (current day)

The −3d, −2d, and −1d columns in the Recent panel represent completed UK calendar days. Apply the pollutant-specific rules above: PM₂.₅ and PM₁₀ daily means, NO₂ maximum hourly index, and O₃ maximum rolling-8h index.

The Today column is different. While the day is still in progress, PM₂.₅, PM₁₀, and O₃ do not use the completed-day formulas alone — they rely on a trigger system that predicts an index level from rising concentrations as hourly data arrives. This logic is implemented in the Awair project (separate from this prototype).

NO₂ for today still follows the standard hourly rule: take the maximum hourly index across UK-local hours received so far (00:00 through the latest complete hour).

Timestamps and UK calendar days

The exposure service returns hourly records where each timestamp is the start of that hour, always in GMT (UTC). The widget, however, labels days as UK calendar dates (−3d, −2d, −1d, Today) — midnight to midnight in UK local time.

During BST (British Summer Time, roughly late March to late October), UK local time is UTC+1. A single UK calendar day therefore spans two GMT dates. You must not group records by GMT date alone.

BST example — UK 6 July 2026 (a summer day):

UK-local 6 Jul 00:00–23:00 corresponds to these GMT timestamps (time beginning):

  • UK 00:00 → 2026-07-05T23:00:00Z
  • UK 01:00 → 2026-07-06T00:00:00Z
  • …
  • UK 23:00 → 2026-07-06T22:00:00Z

To compute the PM daily mean for “6 Jul”, select exactly these 24 hourly records — not all records where the GMT date is 6 Jul. The same hour selection applies when taking NO₂ hourly maxima or O₃ rolling-8h maxima for that day. For O₃ at UK 00:00 on 6 Jul, the rolling window also needs UK-local hours from 17:00–23:00 on 5 Jul — i.e. 2026-07-05T16:00:00Z through 2026-07-05T22:00:00Z during BST.

Implementation checklist:

  1. Convert each API timestamp from GMT to UK local time (use Europe/London — handles GMT/BST automatically).
  2. Assign each hour to a UK calendar date based on its local time.
  3. For each UK date, collect the 24 values at local hours 00:00–23:00. PM₂.₅/PM₁₀: daily mean. NO₂: max hourly index. O₃: max rolling-8h index (using hours from prior local days where needed).
  4. When requesting data, extend the GMT from/to window: one day on each side during BST for day grouping, plus at least seven hours before UK 00:00 on −3d for the first O₃ rolling window.

Outside BST (GMT = UK local), UK midnight aligns with GMT midnight and grouping is straightforward — but always use UK-local assignment so the logic works year-round.

Step 2 — Round, then map to DAQI index

Before comparing any concentration to DAQI bands, round it to the nearest whole µg/m³ (standard rounding — values ending in .5 round up). This applies to all pollutants and all calculation types: PM daily means, NO₂ hourly values, O₃ rolling 8-hour means, and any other value mapped to levels 1–10. Per DEFRA implementation guidance, rounding must be the last step — compute means from unrounded hourly values first, then round once immediately before band lookup. Do not round intermediate hourly values or round more than once.

Rounding example (illustrates the general rule, not an O₃-only special case):
  • 101.4 µg/m³ → rounds to 101 before band comparison
  • 101.5 µg/m³ → rounds to 102 before band comparison

The same rule applies at every threshold boundary — e.g. NO₂ 67.4 → 67, 67.5 → 68; PM₂.₅ 11.4 → 11, 11.5 → 12.

Compare each rounded value against the lower-bound thresholds for that pollutant. The index is the highest band whose threshold the rounded value meets or exceeds (levels 1–10). Gov.uk publishes these as integer ranges (e.g. O₃ level 3 = 67–100, level 4 = 101–120); the lower-bound table below is equivalent.

Step 3 — Take the highest index across pollutants

For each day, the displayed DAQI level is the maximum index across PM₂.₅, PM₁₀, NO₂, and O₃ — using each pollutant’s Step 1 result (PM daily mean; NO₂ max hourly; O₃ max rolling-8h). This follows the UK approach of reporting the pollutant driving the overall band.

dayLevel = max(pmDayIndex, pm10DayIndex, no2DayIndex, o3DayIndex)

Step 4 — Render the ladder

Display a 10-segment vertical ladder per day. Fill rule: segments 1 through dayLevel are coloured; segments above dayLevel are grey. Show the DAQI key strip below the Recent panel (Low · Moderate · High · V.High).

Long-term bar mathematics (Designs 3.1 / 3.2)

The long-term panel compares annual mean concentrations to WHO air quality guidelines. It uses a separate scale from DAQI — do not mix annual WHO comparison with daily index levels.

WHO guidelines (µg/m³)

PollutantWHO annual guideline
PM₂.₅5
PM₁₀15
NO₂10
O₃60

Bar height

The WHO dotted line is fixed at 52% of the bar track height.

Below the line (concentration up to the WHO guideline):

if annual ≥ WHO: belowHeight = 52% of track else: belowHeight = (annual / WHO) × 52%

Above the line (excess over WHO):

excessRatio = (annual − WHO) / WHO aboveHeight = (excessRatio / maxExcessAcrossPollutants) × 48% of track

The pollutant with the largest excess ratio fills the zone above the line; others scale proportionally.

Display conventions

  • Show annual concentrations as integers (µg/m³)
  • Show ratio as 3.6× — no “WHO” suffix on the multiplier
  • Label the year from annualYear (e.g. 2022)
  • Use pollutant bar colours (solid above WHO, light fill below); names in the key strip below the panel

Panel-by-panel summary

Long-term

  • Data: value per species from the same range call (LAEI 2022 annual mean at lat/lng); label with annualYear: 2022
  • Render: four vertical bars (PM₂.₅, PM₁₀, NO₂, O₃), shared WHO dotted line at 52%, ratio above bar, concentration below
  • Key strip: pollutant colour swatches + names below the panel

Recent

  • Data: nowcast_value from the same range call; group into UK-local calendar days (−3d through today), adjusting for BST
  • Process: group into UK-local days (BST-aware) → pollutant-specific DAQI inputs (PM daily mean; NO₂ max hourly; O₃ max rolling-8h) for −3d/−2d/−1d; today uses additional Awair trigger logic for PM₂.₅, PM₁₀, and O₃ (see Step 1) → round and band per pollutant → max across pollutants
  • Render: four day columns (−3d, −2d, −1d, Today) with 10-segment DAQI ladders
  • Key strip: DAQI band legend below the panel

Forecast

  • Data: text band from London Air forecast
  • Process: map band to display index (Low→3, Moderate→6, High→9, V.High→10)
  • Render: single DAQI ladder for tomorrow; band name and date below
  • Credit: ERG logo below the panel
